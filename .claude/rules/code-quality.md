# 코드 품질 및 보안 표준

## CODE QUALITY STANDARDS
- 중복을 철저히 제거
- 명명과 구조로 의도를 명확하게 표현
- 의존성을 명시적으로 만들기
- 메서드를 작고 단일 책임에 집중
- 상태와 부작용 최소화
- 가능한 가장 간단한 솔루션 사용

## REFACTORING GUIDELINES
- 테스트가 통과하는 "Green" 단계에서만 리팩토링
- 확립된 리팩토링 패턴을 정확한 이름으로 사용
- 한 번에 하나의 리팩토링 변경만 수행
- 각 리팩토링 단계 후 테스트 실행
- 중복 제거 또는 명확성 향상 리팩토링 우선

## Security
- 하드코딩된 시크릿 금지
- 민감한 데이터는 환경 변수 사용
- 모든 사용자 입력 검증
- 파라미터화된 쿼리만 사용
- CSRF 보호 활성화
